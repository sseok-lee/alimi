## 1. 문제 정의

### 1.1 현재 상황 (As-Is)

국민들이 받을 수 있는 정부 지원금과 복지 혜택이 많지만, 정보가 흩어져 있어 찾기 어렵습니다.

- 중앙정부, 지자체별로 수십 개의 웹사이트가 따로 운영됨
- 복잡한 자격 조건으로 내가 받을 수 있는지 판단하기 어려움
- 블로그/커뮤니티에서 과도한 정보량으로 인해 오히려 혼란스러움
- 신청 기한을 놓쳐서 지원금을 못 받는 경우 발생

### 1.2 목표 상황 (To-Be)

"복지알리미"에 들어와서 나이, 소득, 거주 지역 3가지만 입력하면:
- 내가 받을 수 있는 지원금이 자동으로 추천됨
- 불필요한 정보는 걸러지고 정확한 정보만 표시
- 예상 수령 금액까지 바로 확인 가능

마치 "구글 검색"처럼 간단하지만, "정부 지원금 전문 검색엔진"처럼 정확합니다.

### 1.3 왜 지금인가? (Why Now)

- **구글 애드센스 수익 창출**: 금융/복지 키워드는 CPC가 높아 수익성이 우수함
- **정보 제공의 사회적 가치**: 정보 불균형 해소로 국민의 경제적 부담을 줄일 수 있음
- **공공데이터 활용 경험**: 보조금24 API 등 공공 API가 점점 더 접근하기 쉬워짐

## 2. 사용자 페르소나

### 2.1 주요 페르소나

| 항목 | 내용 |
|------|------|
| 이름/별칭 | "지원금이 필요한 민수" |
| 나이/직업 | 35세, 프리랜서 디자이너 |
| 핵심 니즈 | 경제적 부담 감소, 맞춤형 지원금 찾기 |
| 불편 포인트 | 지원금 정보가 여러 사이트에 흩어져 있어 찾기 어려움, 자격 조건 판단 어려움 |
| 사용 상황 | 필요할 때마다 지원금 검색, 자격 조건 충족 여부 즉시 확인 |

### 2.2 페르소나의 하루 (시나리오)

민수씨는 프리랜서로 일하며 불규칙한 수입에 어려움을 겪고 있습니다. 문득 "프리랜서 지원금"을 검색해봅니다.

구글에서 "복지알리미"를 발견하고 클릭합니다.
- **나이**: 35세
- **소득**: 연 3,000만원 (프리랜서)
- **지역**: 경기도 성남시

3가지만 입력하고 "검색" 버튼을 누르자, 즉시 화면에 나타납니다:
1. **소상공인 경영안정자금**: 예상 수령액 최대 5,000만원
2. **고용보험료 지원**: 월 최대 10만원
3. **국민취업지원제도**: 월 최대 50만원

민수씨는 "바로 이거야!" 하며 각 지원금의 신청 링크를 클릭합니다. 5분 만에 모든 정보를 확인하고, 바로 신청 절차로 넘어갑니다.

## 3. 사용자 스토리

### FEAT-0: 공통 흐름 (온보딩/검색)
- [ ] 사용자로서, 별도 가입 없이 바로 검색할 수 있어야 한다. 왜냐하면 가입 절차가 귀찮으면 이탈하기 때문이다.
- [ ] 사용자로서, 모바일에서도 동일하게 사용할 수 있어야 한다. 왜냐하면 대부분의 사용자가 모바일로 접근하기 때문이다.

### FEAT-1: 나이/소득/지역 입력 맞춤형 지원금 매칭 (MVP)
- [ ] 사용자로서, 나이/소득/지역 3가지만 입력하면 자동으로 나에게 맞는 지원금이 표시되어야 한다. 왜냐하면 복잡한 조건 입력은 포기하게 만들기 때문이다.
- [ ] 사용자로서, 예상 수령 금액을 즉시 확인할 수 있어야 한다. 왜냐하면 "받을 수 있는 돈"이 명확해야 신청 동기가 생기기 때문이다.
- [ ] 사용자로서, 지원금별 자격 조건을 쉬운 말로 확인할 수 있어야 한다. 왜냐하면 전문 용어는 이해하기 어렵기 때문이다.
- [ ] 사용자로서, 신청 링크를 바로 클릭할 수 있어야 한다. 왜냐하면 다시 검색하는 번거로움을 없애야 하기 때문이다.

### FEAT-2: 신청 기한 알림 (v2 이후)
- [ ] 사용자로서, 이메일로 신청 기한을 알림받고 싶다.

### FEAT-3: 지원금 통합 검색 (v2 이후)
- [ ] 사용자로서, 키워드로 지원금을 검색하고 싶다.

|--------|------|--------|

| 검색 전환율 | 사이트 방문 후 검색을 실행한 비율 | 70% 이상 |
| 페이지 체류 시간 | 사용자가 사이트에 머무는 평균 시간 | 2분 이상 |

## 5. 범위 관리

### 5.1 MVP (반드시 포함)
- FEAT-1: 나이/소득/지역 입력 맞춤형 지원금 매칭
- FEAT-0: 기본 랜딩 페이지 + 검색 결과 페이지

### 5.2 옵션 (시간 허용 시)
- 지역별 상세 필터링 (구/동 단위)
- 예상 수령 금액 계산기 고도화

### 5.3 Non-goals (명시적 제외)
- **소셜 로그인/회원가입**: 가입 없이 바로 검색 가능하도록 MVP는 익명 사용
- **이메일 알림 서비스**: v2에서 추가 예정
- **커뮤니티/댓글 기능**: 정보성 사이트에 집중, 커뮤니티는 제외
- **모바일 앱**: 반응형 웹으로 모바일 대응

## 7. 해결 가설과 실험

### 7.1 가설
> 만약 나이/소득/지역 3가지만 입력하면 맞춤형 지원금을 보여주면, 사용자의 검색 전환율이 70% 이상 나올 것이다. 왜냐하면 간단한 입력으로 정확한 결과를 받을 수 있기 때문이다.

### 7.2 실험 루프

```
실험 → 관측 → 학습 → 다음 가설
```

| 단계 | 내용 |
|------|------|
| 실험 | MVP 출시 후 2주간 DAU 및 검색 전환율 측정 |
| 관측 | Google Analytics로 유입 경로, 체류 시간, 검색 클릭률 측정 |
| 학습 | 검색 전환율이 목표(70%)에 도달했는지 판단 |
| 다음 | 목표 미달 시 입력 필드 수 줄이기, UI 개선 실험 진행 |
